# Website Visual Test v0.1.0

Release date: 2026-07-01

## Overview

This is the first versioned release of Website Visual Test, a local Playwright-powered visual comparison tool for validating the same page paths across two website domains. The release is designed for both technical and non-technical users: engineers can run the CLI workflow, while reviewers can use the local dashboard to extract URLs, run comparisons, watch progress, review results, and export report bundles.

## Highlights

- Local dashboard at `http://127.0.0.1:4317/` for non-technical users.
- Baseline vs target domain comparison across desktop, tablet, and mobile viewports.
- URL discovery from sitemap, browser crawl, combined sitemap plus crawl, or a pasted manual relative URL list.
- Manual relative URL workflow for validating a known path list across multiple domains.
- Live progress bar, current-step label, and timestamped job log.
- Live Results table that updates while comparisons are still running.
- HTML report with screenshot previews and zoomable lightbox.
- Markdown summary for sharing and review workflows.
- Past Results history in the dashboard.
- ZIP export for the latest report or any historical run.
- Local project presets stored outside git so client URLs are not committed.
- First-time setup helper and dashboard preflight checks.
- Node.js compatibility for Node 18, 20, and 22.
- GitHub Actions compatibility matrix for Linux, Windows, and macOS.

## User Workflows

### Dashboard Workflow

1. Run `npm run setup` the first time on a machine.
2. Run `npm run visual:ui`.
3. Open `http://127.0.0.1:4317/`.
4. Enter the baseline and target URLs.
5. Choose a URL source:
   - `Sitemap` for sitemap-driven discovery.
   - `Crawl` for same-origin browser discovery.
   - `Both` to merge sitemap and crawl discovery.
   - `Manual List` to paste a known path checklist.
6. Run `Extract URLs`, `Compare`, or `Extract + Compare`.
7. Watch progress, logs, and live result rows while the job runs.
8. Open the final report or download the ZIP bundle.

### Manual URL List Workflow

Manual URL lists are useful when discovery is noisy or when stakeholders provide a fixed validation checklist.

Accepted entries include:

- `/`
- `/about`
- `contact`
- full URLs copied from another domain

The tool keeps the path and rebuilds matching baseline and target URLs from the domains entered for the run.

CLI example:

```sh
npm run visual:urls -- --baseline=https://example.com --target=https://www.example.com --url-source=manual --manual-urls=/,/about,/contact
```

### CLI Workflow

```sh
npm run setup
npm run visual:urls -- --baseline=https://example.com --target=https://www.example.com --url-source=sitemap --max-pages=100
npm run visual:test -- --baseline=https://example.com --target=https://www.example.com
npm run visual:report
```

Full workflow:

```sh
npm run visual:all -- --baseline=https://example.com --target=https://www.example.com
```

## Reports And Outputs

Run outputs are written to:

```text
visual-test-results/YYYY-MM-DD/run-###/
```

The latest completed run is also available at:

```text
visual-test-results/latest/
```

Generated artifacts include:

- `index.html` - reviewer-friendly visual report.
- `summary.md` - Markdown summary.
- baseline screenshots.
- target screenshots.
- diff images.
- per-comparison metadata.
- `run.json` with full run data.

URL extraction outputs include:

- `visual/urls.json`
- `visual/urls.md`
- `visual/pages.json`
- `visual/pages.md`

## Compatibility

Supported runtime:

- Node.js 18 or newer.

Validated compatibility matrix:

- Node 18, 20, and 22.
- Linux, Windows, and macOS through GitHub Actions.

Command compatibility notes:

- npm scripts use `tsx` directly for compatibility across supported Node versions.
- Windows users can set the dashboard port with PowerShell or Command Prompt syntax documented in the README.

## Privacy And Repo Hygiene

- Client-specific URLs are not required in tracked files.
- Saved project presets are written to `visual/presets.json`, which is ignored by git.
- Generated reports and screenshots are ignored by git.
- The repository defaults use `example.com` values.

## Validation

This release was validated with:

- `npm test`
- local dashboard preflight
- manual URL extraction through CLI and API
- dashboard DOM verification
- live-results API verification with a deliberately slow local comparison
- GitHub Actions Node Compatibility workflow

## Known Notes

- Final HTML and Markdown reports are generated after the comparison job completes.
- Live Results rows are visible during a running job, including screenshot evidence links as each comparison finishes.
- The dashboard is intended for local trusted use and binds to `127.0.0.1` by default.
